## 1. 用户故事拆分

### 用户故事：作为超声模拟软件用户，我需要加载STL格式的病人心脏模型文件，以便在3D场景中查看和分析

**拆分后的子任务：**

#### 1.1 数据层用户故事
- **故事1.1**: 作为系统，我需要解析STL文件格式，以便将二进制数据转换为3D网格
- **故事1.2**: 作为系统，我需要提供文件系统抽象，以便在不同平台上读取文件

#### 1.2 业务逻辑层用户故事  
- **故事2.1**: 作为模型管理器，我需要管理3D模型的生命周期，包括加载、缓存和释放
- **故事2.2**: 作为场景管理器，我需要控制3D场景中的模型显示和隐藏

#### 1.3 表示层用户故事
- **故事3.1**: 作为用户，我需要通过界面选择STL文件，以便加载特定病人的心脏模型
- **故事3.2**: 作为用户，我需要看到加载进度和状态，以便了解操作结果
- **故事3.3**: 作为用户，我需要在3D场景中看到加载的心脏模型，以便进行可视化分析

#### 1.4 状态管理层用户故事
- **故事4.1**: 作为应用，我需要管理模型加载状态，包括加载中、成功、错误等状态

## 8. 文件大小限制调整方案

### 8.1 问题背景

用户反馈遇到247.06MB的心脏模型文件被系统拒绝的问题，当前的文件大小限制为50MB，无法满足实际使用需求。

### 8.2 实施目标

1. **提高文件大小限制**：从50MB提高到250MB，支持大多数心脏解剖模型文件
2. **添加用户确认机制**：对于超过特定阈值（150MB）的文件，显示确认对话框
3. **改进用户体验**：提供更友好的错误消息和性能影响说明
4. **保持系统安全性**：通过确认机制提醒用户大文件可能影响性能

### 8.3 技术实现方案

#### 8.3.1 文件大小限制调整

**当前实现**：
```typescript
// FileSelector.tsx 中的当前实现
const maxSize = 50 * 1024 * 1024; // 50MB
```

**调整后实现**：
```typescript
// 新的文件大小限制
const maxSize = 250 * 1024 * 1024; // 250MB
const confirmationThreshold = 150 * 1024 * 1024; // 150MB确认阈值
```

#### 8.3.2 用户确认机制

**新增组件**：
- `ConfirmationDialog`：确认对话框组件
- 包含大文件警告信息、性能影响说明、确认/取消按钮

**状态管理**：
```typescript
// FileSelector 状态扩展
interface FileSelectorState {
  selectedFile: File | null;
  showConfirmation: boolean;
  pendingFile: File | null;
}
```

**确认流程**：
1. 用户选择文件
2. 文件大小检查：
   - < 150MB：直接加载
   - 150MB - 250MB：显示确认对话框
   - > 250MB：显示错误消息
3. 用户确认后继续加载

#### 8.3.3 错误处理和用户提示改进

**错误消息优化**：
- 当前：`文件大小超过限制: 247.06MB > 50MB`
- 改进：`文件大小超过限制 (247.06MB)。最大支持250MB的文件。`

**新增用户提示**：
- 在文件选择前显示当前文件大小限制
- 添加性能影响说明
- 提供加载进度反馈
